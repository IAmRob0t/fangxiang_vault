### 2. 核心任务与流程

这是一个多步骤任务，请严格按照以下流程执行：

1.  **提取文本：**
    *   编写并执行一个临时的 Python 脚本，使用 `python-pptx` 库直接从 `.pptx` 文件中提取所有文本内容。

2.  **提取图片：**
    *   在与源 PPTX 文件相同的目录下，创建一个名为 `attachments` 的文件夹。
    *   编写并执行一个临时的 Python 脚本，使用 `python-pptx` 库从 `.pptx` 文件中提取所有图片，并保存到上述 `attachments` 文件夹中。

3.  **分析与重命名图片：**
    *   **理解内容：** 逐一分析 `attachments` 文件夹中的每张图片，理解其核心内容（如图表、流程图、代码等）。
    *   **重命名：** 根据图片内容，为其生成一个清晰、有描述性的中文文件名（例如：“ARM架构图.png”）。删除内容重复的图片。

4.  **生成 Markdown 笔记：**
    *   **整合内容：** 将第一步提取的文本和第三步重命名后的图片进行整合。
    *   **保存位置：** 将最终的 Markdown 笔记保存在与源 PPTX 文件相同的目录下，文件名与源文件主名相同（例如：`MyPresentation.md`）。

5.  **清理文件：**
    *   完成所有操作后，删除临时的 Python 脚本和原始的 `.pptx` 文件。

---

### 3. 格式化与排版要求

在生成最终的 Markdown 笔记时，请遵循以下排版规则：

1.  **结构化：** 根据从幻灯片提取的文本逻辑，创建清晰的章节和段落。使用多级标题（`#`, `##`, `###`）来组织内容。
2.  **精简语言：** 对提取的文本进行润色和精简，使其更连贯、更易读，同时保留所有关键信息。
3.  **图文融合：** 将重命名后的图片以 Obsidian 的 `![[图片文件名.png]]` 格式，插入到笔记正文中与其内容最相关的段落附近。
4.  **最终标题调整（最后一步操作）：**
    *   **删除主标题：** 移除笔记最顶部的第一个一级标题 (`#`)。
    *   **提升所有标题等级：** 将笔记中所有剩余的标题等级提升一级（即 `##` 变为 `#`，`###` 变为 `##`，以此类推）。
