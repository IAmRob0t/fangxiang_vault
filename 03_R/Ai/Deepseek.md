## 3. 错误码

您在调用 DeepSeek API 时，可能会遇到以下错误。这里列出了相关错误的原因及其解决方法。

| 错误码            | 描述                                                                                                                          |
| -------------- | --------------------------------------------------------------------------------------------------------------------------- |
| 400 - 格式错误     | **原因**：请求体格式错误  <br>**解决方法**：请根据错误信息提示修改请求体                                                                                 |
| 401 - 认证失败     | **原因**：API key 错误，认证失败  <br>**解决方法**：请检查您的 API key 是否正确，如没有 API key，请先 [创建 API key](https://platform.deepseek.com/api_keys) |
| 402 - 余额不足     | **原因**：账号余额不足  <br>**解决方法**：请确认账户余额，并前往 [充值](https://platform.deepseek.com//top_up) 页面进行充值                                  |
| 422 - 参数错误     | **原因**：请求体参数错误  <br>**解决方法**：请根据错误信息提示修改相关参数                                                                                |
| 429 - 请求速率达到上限 | **原因**：请求速率（TPM 或 RPM）达到上限  <br>**解决方法**：请合理规划您的请求速率。                                                                       |
| 500 - 服务器故障    | **原因**：服务器内部故障  <br>**解决方法**：请等待后重试。若问题一直存在，请联系我们解决                                                                         |
| 503 - 服务器繁忙    | **原因**：服务器负载过高  <br>**解决方法**：请稍后重试您的请求                                                                                      |

# 1. 提示库

## 1. 提示词生成专家

你是一位大模型提示词生成专家，请根据用户的需求编写一个智能助手的提示词，来指导大模型进行内容生成，要求：
1. 以 Markdown 格式输出
2. 贴合用户需求，描述智能助手的定位、能力、知识储备
3. 提示词应清晰、精确、易于理解，在保持质量的同时，尽可能简洁
4. 只输出提示词，不要输出多余解释

## 2. 深度思考

1. **分阶段思考法**（替代批判性思考轮次）
	建议提示词：
	"请分三个阶段分析这个问题：
	第一阶段：列举所有相关基础事实和核心概念
	第二阶段：评估不同角度的潜在关联和矛盾点
	第三阶段：综合各维度信息形成最终结论
	每个阶段请至少展开 5 个分析点"
2. **对抗性论证法**（替代反面考虑轮次）
	建议提示词：
	"请按照以下结构回答：
	1. 陈述初始结论
	2. 提出三个可能反驳该结论的论点
	3. 针对每个反驳论点进行验证：
	   A) 若反驳成立需满足哪些条件
	   B) 现有证据是否支持这些条件
	   C) 是否需要修正原始结论
	4. 最终修订版结论"
3. **多维复盘法**（替代简单复盘轮次）
	建议提示词：
	"请完成回答后额外进行：
	1. 可信度检查：标记结论中概率低于 90%的推断
	2. 盲点检测：列出可能被忽视的 3 个相关领域
	3. 极端测试：假设关键参数变化±50%会如何影响结论
	4. 时间维度：预测该结论在 1 年/5 年后的有效性"
4. **复合型提示模板**：
	"请用学术论文评审标准分析这个问题，依次完成：
	1. 文献综述环节：归纳现有理论框架（至少 3 个）
	2. 方法论检查：验证推导过程的逻辑漏洞
	3. 数据验证：区分事实依据与假设推断
	4. 创新性评估：指出结论的新颖性和局限性
	5. 应用路线图：提出三个实践应用场景及风险"
