# C语言中的进制表示与硬件关联

> 在单片机开发中，理解C语言中的进制表示及其与底层硬件的关联至关重要。

# 1. 进制与硬件基础

计算机底层硬件（如单片机）的运作与进制紧密相关：
*   **二进制 (Binary)**：硬件最基础的语言。晶体管的开关状态直接对应二进制的 `0` 和 `1`。
*   **总线宽度**：处理器和内存之间的总线宽度（如8位、16位、32位）直接决定了数据处理能力和最大寻址范围，这些都是基于二进制的位操作。
*   **内存地址 (Hexadecimal)**：内存地址通常以十六进制表示，因为它能简洁高效地表示大量的二进制位，且每位十六进制数对应四位二进制数，便于理解和操作底层硬件。例如，STM32F1的SRAM区域可能从 `0x20000000` 到 `0x2000FFFF`。

# 2. C语言中的进制表示

C语言支持多种进制的字面量表示，方便开发者直接在代码中使用：

*   **十进制 (Decimal)**：日常使用的数字，直接书写。
    *   示例：`int a = 123;`
*   **十六进制 (Hexadecimal)**：以 `0x` 或 `0X` 开头。常用于表示内存地址、寄存器值等。
    *   示例：`int b = 0x1A;` (相当于十进制的26)
    *   十六进制数字：`0-9` 和 `A-F` (A=10, B=11, C=12, D=13, E=14, F=15)。
*   **八进制 (Octal)**：以 `0` 开头。
    *   示例：`int c = 012;` (相当于十进制的10)
    *   八进制数字：`0-7`。
*   **二进制 (Binary)**：以 `0b` 或 `0B` 开头。
    *   示例：`int d = 0b1010;` (相当于十进制的10)
    *   **注意**：`0b` 前缀是C99标准后的扩展，**C89/C90 标准** 不支持此二进制字面量。在较旧的编译器或严格遵循旧标准的项目中可能无法使用。
