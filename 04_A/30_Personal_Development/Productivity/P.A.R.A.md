# 引言：如何阅读本书

## 1. 如何阅读本书

本书的每个字都旨在做一件事：推动你采取行动。  

阅读时，请划出任何让你觉得有趣、惊讶或特别相关的内容。这些划出的内容可以成为你新 PARA 系统的首批素材！  

我将本书分为三部分，第一部分已包含你需要开始实践的全部知识。我强烈建议你读到这里就停下来，尝试亲自实施 PARA——按照我提供的指南，这只需不到 60 秒。  

第二部分包含我多年指导人们使用 PARA 时总结的额外准则和最佳实践。我建议你设置提醒，在真实世界中测试系统几周后再回来阅读这部分。

阅读本书的唯一回报来自将所学付诸实践，而读完前五章你就能做到这一点。

# 第一部分：PARA 的基础  

## 1. 认识 PARA  

无论你是想保存阅读书籍的摘录、关于新想法的语音备忘录、播客访谈中的励志语录、有用在线资源的网页书签、工作会议或电话笔记、唤起珍贵回忆的照片，还是个人日记，这个系统都将为你提供保存任何信息的工具。不仅是保存，还能巧妙利用它们实现你设定的任何目标。  

### 1.1. **涵盖你整个生活的四类信息**  

PARA 基于一个简单观察：你的所有信息只需四类即可涵盖。  

> [!NOTE] 项目（PROJECTS）
> 你当前正在进行的短期工作或生活目标

> [!example] 领域（AREAS）
> 你需要长期管理的职责

> [!tip] 资源（RESOURCES）
> 未来可能有用的主题或兴趣

> [!success] 存档（ARCHIVES）
> 前三类中的非活跃内容

- 你正在积极进行的**项目**——无论是工作还是生活中的短期努力，都有明确目标。例如：  
	- 完成网页设计  
	- 购买新电脑  
	- 撰写研究报告  
	- 翻新浴室  
	- 完成西班牙语课程  
	- 布置新客厅家具  

- 你负责的**领域**——工作和生活中需要持续关注的广泛责任。例如：  
	- 工作职责：市场营销、人力资源、产品管理、研发、直接下属、软件开发  
	- 个人职责：健康、财务、子女、写作、汽车、家庭  

- **资源**包括你感兴趣或正在学习的各类主题，例如：  
	- 平面设计  
	- 有机园艺  
	- 网页设计  
	- 日本料理  
	- 摄影  
	- 营销素材  

- **存档**包含前三类中不再活跃但可能需要未来参考的内容：  
	- 已完成或搁置的项目  
	- 不再活跃或相关的领域  
	- 不再感兴趣的资源  

### 1.2. 核心原则——根据项目和目标组织信息 

大多数人最初在学校学习如何组织信息。我们被教导按学科（如数学、历史、化学）分类课堂笔记、讲义和学习材料。  

不知不觉中，我们将这种方法带入成年，继续根据“市场营销”“心理学”“商业”“创意”等宽泛主题分类文档和文件。  

这在职后生涯中毫无意义。职场上没有课程、考试、分数或文凭，也没有老师告诉你期末考试要记什么，因为根本没有期末考试。  

你在工作和生活中拥有的是**想要实现的成果**。你可能想推出新产品、做出关键决策，或达成季度销售目标；也可能努力规划家庭旅行、发表个人文章，或在社区找到负担得起的日托服务。  

在忙碌的日常中，当你试图完成这些事情时，绝对没有时间在“心理学”这样庞大的类别中翻找六个月前保存的文章。  

与其像学校那样按宽泛主题组织信息，我建议你根据当前承诺的项目和目标来组织。  

> [!important] 这就是“为行动而组织”的含义，也是本书反复强调的核心理念  

例如，当你坐下来进行平面设计项目时，需要所有相关笔记、文档、素材和其他材料集中一处、随时可用。  

这看似显而易见，但我发现大多数人做的恰恰相反。人们倾向于将推进进度所需材料分散在十几个不同地方，导致开始工作前要花半小时定位它们。  

如何确保每个项目或目标的相关材料集中一处？答案是一开始就按此组织。这样你就能明确存放和查找的位置。  

当执行愿景所需的全部信息触手可及时，目标实现就近在咫尺。让我们看看当这些障碍消失时，你能取得什么成就。  

## 2. 按项目组织的威力  

> [!important] 项目是“短期努力”，需要有明确截止日期

无论你多聪明或多有动力，若不将责任领域分解为具体项目，就无法做到两件关键之事。  
1. 你无法真正了解承诺的规模  
2. 你无法将当前努力与长期目标关联  

### 2.1. 为理想生活而组织  

使用 PARA 不仅仅是创建一堆文件夹来放置东西。它是关于识别你工作和生活的结构——你所承诺的、你想要改变的，以及你想要去的地方。它是关于以一种支持并召唤你想要的未来生活的方式来组织信息。
我们所称的“组织”中的很多内容本质上是伪装的拖延。我们告诉自己我们在“准备”或“做研究”，假装这意味着进展。

实际上，我们正在寻找任何可以打磨或整理的小事，以避免面对我们所畏惧的任务。

PARA 通过这个外表，给我们提供了一种组织任何事物的方法，这种方法简单到没有借口，除了下一步必要的步骤外，没有其他事情可做。
这是一种极简的方式，给你的环境增添足够的秩序，让你有清晰的思路向前推进，仅此而已。
在接下来的几章中，让我们更深入地探讨一下使用这种方法来转变你的数字生活的样子。

## 3. 60 秒 PARA 设置指南  

你可能认为实施 PARA 的第一步是精心创建四类文件夹，然后将现有文件逐个移入。过去，我会花数小时帮客户这样做，但后来发现这是错误的开始方式。在创建新内容前，必须先清理旧内容。 

本章将带你完成在任意数字平台采用 PARA 的三步骤：  
1. 归档现有文件  
2. 创建项目文件夹  
3. 按需创建其他文件夹  

### 3.1. 步骤一：归档现有文件  

在物理世界，每张纸、文件夹和物品都占据宝贵空间，因此你必须决定如何处理每一件物品，即使决定只是扔掉它。  

但数字世界不同。数字物品不占物理空间，只占数字空间（如今几乎是无限的），这意味着你无需删除任何内容，可以保留一切。  

这看似是福音，实则是诅咒。  

保留一切的代价是迅速消耗比物理空间更稀缺的资源：你的注意力。每次看到散落在电脑桌面、文档文件夹、云盘或笔记应用中的杂乱文件，你的心智能量都会被消耗。  

你或许以为合上笔记本就能忽略，但大脑的某个部分会持续担忧数字环境的混乱，直到将其整理有序。对大脑而言，信息环境与物理环境同样重要，只要感到不确定和威胁，它就无法休息。  

你可以保留一切，但不能让其占据注意力中心。它们需要安全且“眼不见心不烦”的存放处——这就是**存档**。将其视为数字生活的“冷存储”，冻结在离开时的状态，确保未来需要时可访问，同时无需担忧。  

具体操作：选中“文档”文件夹中所有现有文件、文档、笔记等（可能有数百甚至数千个），一次性移动到名为“存档 `[今日日期]` 的新文件夹。  

将此日期存档文件夹放入更大的“存档”文件夹，作为未来所有存档的正式归属。  

### 3.2. 步骤二：创建项目文件夹  

第二步：新建名为 Project 的文件夹，作为未来所有项目相关信息的归属。在其中为每个活跃项目创建子文件夹，并以项目名称命名。  

我们接下来关注项目，因为它们是当前正在进行的努力，天然更及时和紧迫。你可以开始将正在处理的文档、笔记或文件移到对应项目文件夹，但无需预先或一次性完成。  

例如，浴室翻新项目可能包含：  
- 浴室尺寸测量  
- 瓷砖图案和颜色的照片  
- 与多位承包商的沟通记录  
- 中标报价和合同  
- 预算追踪表  

即使是外包执行的项目，仍有大量信息需跟踪！  

### 3.3. 步骤三：按需创建其他文件夹  

过去，我会建议在此阶段为预想的领域和资源创建文件夹，认为提前准备会更方便未来存放。后来我意识到这是错误的。  

转折点发生在为硅谷一家软件开发公司整理共享云存储时。我们头脑风暴了所有可能的 PARA 文件夹，并在一天内创建了完整系统。感觉棒极了！但几周后，我听到截然不同的反馈：工程师寻找内容时，总会看到看似完美的空文件夹，导致反复失望。  

> [!warning] 由此我制定新规则
> **永远不要在需要存放内容前创建空文件夹**。  

因此，对于领域和资源，建议暂缓创建“推测性”文件夹，直到确定要存放的内容。创建新文件夹只需片刻，无需提前准备。这两类行动性较低，因此无需在第一天就完善。  

在重组数字生活时，建议借此机会反思未来处理信息的态度。不要随意向新的 PARA 文件夹倾倒内容，否则很快会重回混乱。  

仔细思考 PARA 四类中应保存的内容：
- 什么是真正独特或有用的？
- 专注项目或领域时需要哪些内容？
- 哪些资源真正有价值？
- 哪些可通过谷歌搜索轻松找回？ 

这是清零重启数字生活、基于永恒组织原则的契机。  

腾出的时间和注意力，你将用来做什么？  

## 4. 让组织变轻松的五大关键技巧  

我已介绍 PARA 方法并引导你完成实施步骤。多年来，我在各类事务中使用此系统，发现几个简单技巧能进一步提升其效果。每个技巧只需一次性设置和几分钟实施。

### 4.1. 技巧一：创建收件箱  

繁忙的一周中，你通常无暇完美捕获、命名和归档新内容，因此需要独立处理新内容的时间和空间。  

建议在每个主要平台（如文档文件夹、云存储、笔记应用）的 PARA 四类旁，创建第五个名为“收件箱”的文件夹。  

收件箱是新内容在归位前的临时存放区。下一章将解释如何处理它们。  

### 4.2. 技巧二：为文件夹编号  

建议在五个文件夹名称前添加数字 0-4。用“0”标记收件箱，提醒其内容未处理。按字母排序时，这能保持从“最可行动”到“最不需关注”的顺序。  

大多数时候，你只需查看“1 项目”文件夹；偶尔查看“2 领域”以长期思考；“3 资源”和“4 存档”可安全忽略，直到需要时。  

### 4.3. 技巧三：使用命名约定  

若能一眼看出文件夹属于哪类 PARA，将大有裨益。  

我喜欢用非正式命名约定，例如：  
- 项目文件夹名称以表情符号开头  
- 领域文件夹名称首字母大写  
- 资源文件夹名称小写  

例如，以下明显是项目（表情符号开头）：  
🚀 创建赞助包  
📝 撰写环保新标准文章  
✈️ 计划墨西哥城旅行  

### 4.4. 技巧四：启用离线模式  

### 4.5. 技巧五：备份  

由于你投入大量时间整理数字世界，其价值将大幅提升。为避免努力白费，建议为每个主要平台建立可靠备份系统（因各平台存储不同信息）。  

若使用云平台，此步骤已自动完成（数据存于云端）。电脑文件可使用云备份服务（联网时自动备份）或定期备份至外置硬盘。  

### 4.6. 自定义你的 PARA 系统  

你可能已想象出实践 PARA 后的美好生活图景，我衷心希望它实现。但若中途受挫，请回到此处。生活常忙乱，因此 PARA 设计了几种“优雅失败”的方式。  

首先，文件存放位置无关紧要。若归类太费神或耗时，不必纠结。未来检索信息主要依赖搜索功能，因此精确归类是锦上添花，非必需。  

第 9 章将提到，未来有许多机会在类别间“流动”内容，因此初始存放位置的决定非常宽容，每阶段都有备用方案和安全网。  

其次，PARA 四类并非不可变动。我见过有人添加额外类别（如 S 代表“系统”或 V 代表“价值观”），也有人仅用两类（如“热”文件夹放当前行动项，“冷”文件夹放其他）。  

核心原则是将最可行动、最及时的内容分离并给予主要关注，实现方式无数。无需严格遵循我的建议——我允许你自定义。  

## 5. 如何维护你的 PARA 系统  

我知道你生活忙碌丰富，整理文档并非优先事项，也不该是。我也一样——希望花最少时间调整优化系统。  

因此，建议每周仅用五分钟维护 PARA 系统，只需三步：  
1. 重命名收件箱中的新内容  
2. 将新内容归类至 PARA 文件夹  
3. 更新活跃项目  

### 5.1. 步骤一：重命名收件箱中的新内容  

通常，我每周在收件箱积累 10-20 个新数字内容，如团队会议笔记、读书摘录、网站截图或新想法的语音备忘录。  

这些文件初始名称常为“未命名文档”或“新笔记”。花几秒查看每个保存项，将其重命名为更清晰易懂的标题，能极大提升未来检索效率。  

例如：  
- 2023 年 3 月 2 日与 Clara 的会议笔记  
- 《氧气优势》一书摘录  
- 招聘面试技巧实用网站  
- 关于新在线课程创意的语音备忘录  

名称无需花哨或专业，有时加日期，有时不加。只需用几秒想出简短、简单、易懂的标题。  

注意，你可能需要处理多个收件箱，例如：  
- 文档文件夹中的收件箱（第 4 章建议）  
- 云存储的收件箱  
- 数字笔记应用的收件箱  

### 5.2. 步骤二：将新内容归类至 PARA 文件夹  

处理新内容的第二步是将其移至合适的 PARA 文件夹。这同样只需几秒，思考文档最相关的项目、领域或资源即可。  

批量处理所有“归档”任务，我能在几分钟内轻松处理 10-20 个新内容。由于未来主要依赖搜索找回，存放位置的决定风险低。重阅一周内捕获的新信息，也能提醒后续行动。  

### 5.3. 步骤三：更新活跃项目  

最后，查看项目文件夹，根据过去一周的变化调整：  
- 更名项目以反映新范围或方向  
- 将大项目拆分为更易完成的小项目  
- 归档已完成、搁置、取消或移交他人的项目  
- 重新激活休眠项目并移回“项目”文件夹  

### 5.4. 始终从存档开始  

关于“归档”的最大误解是认为信息将永远消失。  

别把存档视为“创意坟墓”。存档代表你的人生经验总和，是成功与失败中获得的宝贵教训和深刻见解。我保证其中包含未来可复用的有用材料。  

启动新项目、年终总结或更新简历时，存档应是起点。它包含你争取加薪、晋升、赢得客户或提议新计划所需的支持证据。  

我常惊讶于从存档中找到有用材料——重复客户的会议笔记、行业背景研究、设计灵感图片——它们都是我的“知识资本”。  

复用这些知识资产不仅节省大量时间，更让我感觉像站在马拉松中点起跑，而非与众人同一起点。  

# 第二部分：PARA 实战指南  

## 1. 如何区分项目与领域  

二十多岁时，我获得第一份专业工作后不久，读到一本名为《搞定》（*Getting Things Done*）的书。作者戴维·艾伦（David Allen）提出了一套系统化、原则化的个人生产力方法，即 GTD，立即对我的工作方式产生巨大影响。  

GTD 让我像戴上 X 光眼镜，突然能以清晰类别看待信息流，每类设计都让行动更轻松。艾伦提出的定义和区分中，最有力的或许是**项目**与**责任领域**的区分。这两类是你每天接触的最可行动的信息，因此也最重要。  

我注意到许多人对此区分感到困惑。让我们通过 PARA 的视角深入探讨两者。  

### 1.1. 项目：有截止目标的目标  

我对项目的定义是任何具备以下要素的努力：  
1. 可标记“完成”的目标  
2. 期望完成的截止日期或时间框架  

**目标** 简单来说就是“项目试图实现的结果。”它可以是一次成功的员工退修会，一个网站重新设计，或一个孩子的生日派对——你试图在现实世界中实现某种事情，以便你可以标记为“完成”。

**截止日期** 为实现你的目标增加了时间限制。你不希望你的努力永远徘徊，不知道自己是否成功或失败。心中有某种结束点（无论是严格的外部强加还是非正式的自我强加）可以让你问自己：“在剩下的时间里，我能完成多少？” 

### 1.2. 责任领域：一个需要长期维持的标准

虽然项目很重要，但并不是所有事情都是项目。你的工作和生活中有一些方面没有明确的最终目标或截止日期。我们称之为“责任领域”。

责任领域具备：  
1. 需维持的标准  
2. 无限期的结束时间  

与工作相关的领域包括你的工作职责，无论是管理、客户服务、财务分析、战略、辅导、直接报告还是顾问。你在个人生活中也有责任领域，比如你的健康、财务、个人发展和人际关系，这些将以某种形式持续到你生活的尽头。在上述所有例子中，这些领域没有特定的结果需要实现。

没有一个终点线可以让你“完成”你的健康，或“彻底结束”战略，或“勾选”财务作为一个持续关注的问题。责任领域没有目标，而是有一个你试图维持的标准。例如，如果你负责工作中的一个领域，比如领导产品开发，那么你负责的产品就有一个绩效标准（或“质量标准”）。这可能包括提升其速度和性能、快速修复错误，以及批准新更新的发布。

对于你的财务，你的标准可能是按时支付所有账单并满足家庭的需求。对于育儿来说，标准可能是每晚与孩子们共度优质时光，并确保他们始终感受到爱与保护。

维持你的领域是一个持续的过程。它需要深思熟虑和自我意识，以感知你想要什么以及每个领域中缺少什么。一个领域并不是一个需要赢得的奖品，而是一场值得享受的舞蹈。这是日常习惯、有意义的仪式和超越任何特定项目的永恒价值的领域。

### 1.3. 区分项目与领域  

> [!important] 简而言之：**项目会结束，领域会持续**。  

然而，还有一些其他的细微差别你应该注意。每个项目通常都属于一个责任领域。例如：
- 跑马拉松是项目，属于健康领域  
- 出书是项目，属于写作领域  
- 存三个月开支是项目，属于财务领域  
- 计划周年晚宴是项目，属于配偶领域  

尽管项目与领域相关，区分它们至关重要。未能区分是许多挫败的根源。  

若将项目（如写书）视为持续领域而无明确目标，会感到漫无目的；若将领域（如维持体重）视为一次性项目，即使成功减重，也可能因未建立长期习惯而反弹。  

换言之，项目与领域需要完全不同的方法、心态和工具才能成功。正确识别它们是成功的第一步。  

## 2. 如何区分领域与资源  

PARA 中第二重要的区分是领域（A）与资源（R）。  

乍看两者相似，尤其某些主题（如“非营利筹款”或“营养研究”）可能归入任一类。  

这取决于该类别信息对你的意义。  

若你是大学公共卫生教授，负责几门讲座课程，同时还发表自己的研究，那么“营养研究”显然是重要责任领域。  

但若你是同校人类学系学生，营养只是次要兴趣，则它属于资源。  

关键在于区分**直接责任**与**单纯兴趣**。我用大写标题标记领域，小写标记资源，以时刻提醒前者更重要。  

再次强调，混淆这两类会导致大量摩擦与精力浪费。  

### 2.1. 领域：你扮演的角色与承担的职责  

领域是生活中需持续关注以维持质量标准的部分，可视为你在工作和生活中扮演的“角色”或“戴的帽子”。  

工作中的领域包括受聘履行的正式职责（如视频制作、法律事务、客户服务），也可能包括逐渐承担的非正式责任（如公司通讯、导师指导、员工 retreat）。  

个人生活亦然。我们甚至在一天中切换不同角色：配偶、父母、足球教练、邻居、朋友。这些角色虽非正式，仍需承担一定责任。“配偶”文件夹可能包含对方喜爱的餐厅、礼物创意或紧急情况下的健康信息；“足球教练”文件夹可能有训练计划、练习日程和队员联系表。任何需参考或记忆以有效履行角色的内容都值得记录。  

### 2.2. 资源：兴趣、好奇与激情  

资源涵盖你当前感兴趣、好奇或热衷的无数事物。  

资源可包括学习的新技能（如霹雳舞、摄影、高尔夫）、好奇的领域或趋势（如育儿、加密货币、人工智能），以及爱好（如木工、烘焙、弹钢琴）。  

尽管对这些追求充满热情，但我用“资源”一词有特定原因：作为天生好奇者，我有数十甚至数百个想探索的主题，但我知道自己倾向收集过多，成为“数字囤积者”。“资源”一词提醒信息的效用，用“是否有用？”替代“是否有趣？”，迫使我思考这些信息能解决什么问题或克服什么障碍。  

考虑到实用性，资源也可包括“资产”，如库存图片、产品推荐语、代码片段、字体样本，或广告业常用的“素材库”（文案人员保存范例以供参考）。  

### 2.3. 邻域是私人的，而资源是可共享的

责任领域本质上是私人的。与您的健康、财务、个人成长或孩子相关的材料，别人无权过问。例如，在我的健康（区域）文件夹中，我保存血液检查、医生就诊的笔记、医疗账单和疫苗接种记录（这些都是仅与我个人相关的事项）。如果我愿意，我可以随时分享单个项目，但这些类别默认应该保持私密。

资源则截然不同，因为学习和探索新事物本质上是社交活动。有许多情况下，您可能想与其他人分享您的笔记——例如，获取您正在撰写的文章的反馈，与同事比较您们都在学习的技能的笔记，或向访问您曾去过的城市的朋友提供餐厅推荐。因此，我建议您将您的资源文件夹视为“默认可共享”。这样，您可以随时与他人分享单个文档（甚至整个文件夹），而无需先仔细检查其中的任何个人细节。

## 3. 在多平台上扩展 PARA  

PARA 的关键特性之一是普适性——它适用于所有存储信息的平台，任何类型的信息，任何设备均可访问。  

我注意到，即使最有条理的人也常犯致命错误——在每个存储信息的地方使用不同组织系统。待办清单、电脑、云存储、笔记应用各有各的体系。  

这有问题，因为每个系统都有维护和使用所需的认知开销。即使每个方法孤立看合理，组合起来也会产生难以承受的脑力负荷。  

相比之下，PARA 是“平台无关”的单一系统，可实施于任何地方，包括：  
- 待办应用  
- 电脑文件系统（或文档文件夹）  
- 云存储  
- 数字笔记应用  
- 其他信息存储平台  

这为何重要？你永远需要多平台完成工作和生活。大多数项目和领域涉及不同类型的内容，每种需存于适合的平台。  

例如，若你发布某新兴行业研究报告，可能涉及数据（存于电子表格应用）、产品照片（存于图片应用）、专家访谈笔记（存于笔记应用）、行业出版物 PDF（存于文档文件夹）、下一步计划（存于待办应用）。  

技术进步太快，没有单一应用能满足所有需求。与其逆流寻找“全能应用”，不如自由使用多个应用，同时在每个应用中复制相同结构。建议保持拼写、标点、大小写完全一致，以便在不同平台间无缝切换。这样既能利用各平台独特功能，又不牺牲信息处理的一致性与连贯性。  

PARA 完美映射你生活的结构到每个使用的平台。与其让生活适应工具偏好，不如反其道而行：决定你希望如何安排生活与工作，再问工具如何支持它。  

### 3.1. 如何决定存放位置  

由于多平台均有 PARA 类别，接下来的问题是：如何确定某项该存于哪个平台？  

我遵循以下经验法则：  
1. 特定时间的约会或会议 → 日历  
2. 可随时完成的任务 → 待办应用  
3. 文本 → 笔记应用（因其搜索功能最佳）  
4. 需协作的内容 → 云存储  
5. 无法归入上述的内容（如大文件或特殊格式） → 电脑文件系统（文档文件夹）  

注意，上述平台中有四个使用 PARA 组织：  
- 待办应用（仅项目和领域，因只有这两类关联任务）¹¹  
- 笔记应用  
- 云存储  
- 电脑文件系统  

### 3.2. 跨平台系统的优势  

此时有人会问：“我真的需要在所有平台创建对应文件夹吗？”  

答案是：完全不必！  

首先，仅在有内容存放时才在平台创建文件夹，否则会形成空文件夹迷宫，占用空间且无意义。创建新文件夹仅需几秒，因此仅在需要时创建。  

其次，无需在所有平台为同一类别创建对应文件夹。例如，Zoom 通话录音自动存于电脑的资源文件夹，无需在其他平台创建“Zoom 录音”文件夹。许多信息类别可仅存于单一平台——仅在需要推进项目时跨平台扩展。  

我知道手动在不同平台创建相同文件夹看似费时，但回报巨大：你不受限于任何单一平台。  

我曾为某电信公司做大型咨询项目时，使用的项目管理平台突然停运（被收购后关闭）。团队陷入混乱时，我因项目和领域独立于任何软件，几分钟内就迁移到新平台。  

生产力软件格局总在变化，但你的组织方法无需随之改变。若某功能失效或平台政策价格突变，仅影响系统一部分，PARA 确保风险仅限于数字生活的局部，不会波及全部。  

PARA 是跨平台系统，因为你的项目本就跨平台。你很少仅用单一应用完成项目。PARA 提供了一种方式，将存储于多处的信息串联起来。  

试想，若你使用的软件工具和谐协作，而非制造问题，工作将如何改变？  

## 4. 保持信息流动  

开始体验 PARA 威力时，最常见的疑问关于未来：  

“我需要做什么来维持这个系统？”  

这正是 PARA 的闪光点，因为答案是：几乎不用。  

PARA 对信息格式无严格规定——文本、图片、PDF、音视频、幻灯片、GIF 等均可。对命名无规则——描述性标题即可。对文件夹内部组织无要求——子文件夹可按创建日期排序，但电脑可自动处理。  

> [!question] 那我们还需做什么？  
> 答案是：**保持信息流动** 

PARA不是像汽车发动机那样需要非常精确和定期维护的机械系统。它更像是一个有机系统，比如一个池塘或森林。就像一个池塘如果水不流动就会开始停滞并散发气味一样，如果它所包含的知识停止流动，你的PARA系统很快就会过时并失去相关性。

### 4.1. PARA 内的信息流动  

大多数组织方法是静态的：它们假设每个信息都有一个正确的位置。你可能还记得你当地图书馆的杜威十进制系统，精确的索书号告诉你每本书在书架上的确切位置。但当涉及到个人组织数字信息时，并没有这样的“正确”位置。PARA是一个动态系统：任何给定的文件或文档可以放在任意多个地方——重要的是你与它的关系。而这种关系一直在变化。

PARA的内容在不同类别之间不断流动，因为您的需求、目标、生活方式和优先事项在变化。

当今世界唯一不变的是变化，因此我们需要避免鼓励固定思维的僵化系统。  

### 4.2. 保持信息流动的其他例子  

- **项目→领域**：马拉松训练计划（项目）转为日常，移至“锻炼”领域文件夹。  
- **领域→项目**：从“运营”领域收集的创意启动新项目，提升组织运作。  
- **领域→资源**：某城市活动场地列表（原属个人领域）转为可共享资源。  
- **资源→领域**：为改善健康开始居家烹饪，将“食谱”资源移至“烹饪”领域文件夹。  
- **领域/资源→存档**：失去兴趣的爱好（观鸟、国际象棋、柔术、摩托车维修）可归档。  
- **存档→项目**：组织行业会议时，复用过去活动材料。  

最后一点：虽然我更喜欢将笔记和文件整体移动到另一个地方，但实际上你有四种选择来将现有的信息与新类别关联起来：
1. 移动单个项目（如果只有一个项目与新项目相关）
2. 移动一个装满项目的文件夹（如果一整组项目相关）
3. 将两个项目链接在一起（如果你想保持原始项目的位置）
4. 用相同的标签标记项目（如果你想将多个项目相互关联而不移动它们）

我唯一不建议的是行为是**重复**：切勿保留文件的两个版本，否则无法确定哪个最新。  

保持信息流动是为让自己保持行动。当周围信息不断流动变化，你更容易以新视角看待问题，避免陷入困境。甚至会发现项目在不知不觉中已启动！  

## 5. 在团队中使用 PARA  

我们的数字世界常与他人的数字世界碰撞重叠。知识本应共享，因此你可能想知道 PARA 能否用于团队协作。好消息是，PARA 对团队、公司和其他组织的效果与个人同样出色！  

我已经咨询和培训了来自各个行业的数百人，教他们如何组织自己的知识，从像世界银行这样的数十亿美元金融机构，到像基因泰克这样的领先生物技术公司，再到像 Sunrun 这样的创新初创公司。在信息时代，记录和获取员工所了解的知识的能力对生存和发展至关重要。  

PARA 属于“知识管理”（KM）领域，其目标是促进人们有效地共享知识。  

我清楚地记得在我曾工作的一家公司，一位高管有一天突然决定“实施”知识管理。一个维基（或知识库）被创建，我们被告知要通过输入我们所学的内容来“分享我们的知识”。在最初的几周之后，没有人再看过它。那次经历和其他类似的经历让我明白，这种“自上而下”的知识管理方法存在几个主要问题。阐述知识需大量时间精力，由于大多数员工并没有因这种努力而获得报酬或评估，导致努力流失。  
公开分享想法也存在被批评、误解或被替代的风险。目睹自上而下 KM 多次失败后，我得出结论：现代组织需要“自下而上”的方法。知识管理必须以个人需求为中心，主要设计用于提升他们的个人生产力和有效性，以便他们能够发挥出最佳的工作能力。

以下是用 PARA 实现团队 KM 的四大建议：  
1. 明确组织的 PARA 风格  
2. 培训员工使用 PARA  
3. 仅共享项目存于共享平台  
4. 鼓励写作文化  

### 5.1. 建议一：明确组织的 PARA 风格  

我的第一个建议是明确PARA在您组织中的具体表现。即使您决定完全遵循我的建议，PARA总会有一种适合您文化的“风格”。我建议为您的团队创建一个“PARA手册”，其中包括以下决策：
- 如何定义“项目”“领域”“资源”“存档”？  
- 新项目启动的正式流程  
- 项目完成、搁置或取消的标记标准  
- PARA将使用哪些官方支持的平台？
- 使用 PARA 的规则、准则与规范  
- 谁会成为负责监督其实施并确保遵循相关准则的“PARA 冠军”呢？

### 5.2. 建议二：培训员工使用 PARA  

应将 PARA 实施视为培训挑战，而非技术问题。管理者常误以为可“安装”PARA 而无需教学，这大错特错。必须教会团队 PARA 的运作方式及其在组织中的具体应用。  

建议用“PARA 实战手册”进行演示、举办研讨会，确保全员理解。即使面对如此简单的方法，您也会惊讶于大多数人需要摒弃旧习惯与思维定式（如“存在正确组织方式”“信息有唯一归属”“正式结构与精确分类总是更好”）。  

切勿吝啬培训，确保同事理解共享规范与政策。 

### 5.3. 建议三：仅共享项目存于共享平台  

开始创建共享 PARA 时，警惕常见错误：一次性将所有数字资产移入单一共享系统。  

逻辑是：若内容有价值，是否应全员访问？答案是否定的——你绝对不希望每个人随时都能访问所有内容。要理解原因，你必须明白，有效地传达一条知识需要巨大的认知努力。

例如，我曾有某现代项目管理书籍的私人笔记，内容杂乱仅自己理解。想与团队分享时，发现不能简单地通过电子邮件将我的非正式笔记发送给他们，需补充背景、结构、术语解释、思考过程等。这类任务耗费时间精力，与推进优先事项形成权衡。  

因此建议团队默认将个人笔记、文件存于个人 PARA 系统，仅协作项目、领域或资源移入公司共享文件夹。这确保员工仅访问工作所需信息，不多不少。  

### 5.4. 建议四：鼓励写作文化  

很快你会发现，知识管理本质上是一种沟通形式。如《打造第二大脑》所言，文档、笔记是穿越时空发送给未来接收者的信息。信息质量决定其能否被接收和理解。  

高质量沟通满足以下标准：  
- 有趣吸引人（让人想读）  
- 精确清晰（易于理解）  
- 共情（从读者角度撰写）  
- 解决问题（明显有用有效）  
- 激发行动（易于应用）  

这些问题凸显出，有效的 KM 取决于书面表达能力。简言之，共享知识的唯一方式是创建团队书写文化。  

如何做到？以下五法效果显著：  
- **以身作则**：高管和经理定期书面分享重要想法与决策  
- **提供激励**：奖励花时间书面表达思考的员工  
- **给予反馈**：私下反馈下属的草稿后再公开分享  
- **预留阅读时间**：会议以阅读时间开始，强调书面背景的重要性  
- **标准化**：采用内部写作标准（如备忘录、提案、一页纸），创建模板（如 Google 文档或 Notion 页面）  

您提供的鼓励与激励越多，同事越可能坐下来书面表达想法。此习惯将带来更优质思考、更好决策讨论，最终实现更有效的 KM。  

# 第三部分：深度探索  

## 1. 创建项目清单  

如果你聘请我作为你的效率教练，我们最初的几次会议将专注于制定你的「项目清单」。这份清单是你高效生活的核心，没有它，我们几乎无法开展其他工作。  

现在，让我为你省下这笔费用，亲自带你完成这一过程。

你的「项目清单」是你当前承诺要达成的一系列成果的集中体现。它是一份关于你试图完成、创造、实现或解决的所有事情的清单。  

它像待办清单，但规模更大、时间跨度更长，让你看清前进的方向；它像目标清单，但更务实、更扎根于当下。  

制定项目清单本身就是一个极具价值的练习，也是 PARA 方法的基石。大多数人会发现，按照以下步骤操作只需 5 到 10 分钟，最终列出约 10 到 20 个项目。这就是为什么我们要用软件而非大脑来管理它们！  

### 1.1. 步骤 1：列出当前项目  

设定 5 分钟计时（足够完成“第一轮”），根据以下问题写下你想到的任何事项（工作或生活相关均可）：  
- 你目前担忧什么？  
- 哪些问题占用了超出合理范围的精力？  
- 哪些事情需要推进但迟迟没有进展？  
- 你正在采取的行动中，哪些属于尚未明确的大项目？  
- 你想学习、发展、构建、表达、追求、启动、探索或尝试什么？  
- 你想学习哪些技能或培养哪些爱好？  
- 哪些项目能推动你的职业发展或让生活更有趣？  

### 1.2. 步骤 2：为每个项目添加目标  

记住，一个项目必须包含：  
1. 目标  
2. 截止日期（或其他时间框架）  

花一分钟为清单上的每个项目在括号内添加目标。例如：  
- **项目**：就医解决背痛（目标：背痛消除，夜间能无痛入睡）  
- **项目**：制定员工静修议程（目标：团队明确需完成事项并分配后续步骤）  
- **项目**：与 Linda 制定销售活动方案（目标：方案获高管团队批准并分配预算）  

### 1.3. 步骤 3：添加截止日期或时间框架  

再次浏览清单，为每个项目添加完成日期。不必纠结这是严格的“截止日期”还是你期望完成的日期。  

### 1.4. 步骤 4：优先排序清单  

你不太可能在一周内推进清单上的所有项目，甚至大部分都做不到。关键在于仅针对接下来的一周进行优先级排序。  

**下周**，哪些项目应占据你的主要精力？将它们置顶。哪些项目暂时无需关注？将它们移至底部。  

你每周的目标只需是推进清单顶部的少数几个项目。  

### 1.5. 步骤 5：重新评估项目清单  

现在你已清晰列出本周的所有承诺，可以借此机会问自己几个尖锐但极具启发性的问题：  
- 哪些你声称重要的目标或优先事项却没有任何与之相关的项目？（这些被称为“梦想”，因为它们不太可能在短期内实现。）
- 哪些你投入大量时间的项目却没有明确目标？（这些只是“爱好”，因为它们没有目标，可能只是“为了乐趣”。）  
- 哪些项目可以取消、推迟、缩小范围、委派、外包或澄清？  

> [!warning] 注意
> 梦想和爱好本身并无问题，它们是生活的重要部分，但不要将其与项目和责任混为一谈。 

我们只是要让你的时间精力分配到真正重要的事物上。日常忙碌中，二者很容易偏离。不知不觉间，我们便忽视了声称最重要的事，却将宝贵时间投入无关紧要之事。  

上述五个步骤可成为“每周回顾”的一部分。每周执行一次，或在感到不堪重负时随时使用，我保证你将在几分钟内重获清晰与目标感。  

### 1.6. 带着底气说“是”与“否”  

一份准确的项目清单能让你自信地对新承诺说“是”或“否”。一旦清楚自己的实际能力，你便能战略性地决定如何分配精力，而非被动应对。  

我的许多教练客户在完成此练习后发现：他们试图做太多事。看到现有承诺的全貌后，他们意识到自己需要的不是另一个效率工具，而是对不重要的事说“不”。  

当你明确了那些最重要的承诺后，便能冷静决定哪些保留、哪些舍弃。  

每对次优事项说一次“不”，其占据的时间精力便会释放给更重要的事。  
  
## 2. 保持组织的三大核心习惯  

一个残酷的真相是：你很可能很快就会忘记本书的所有内容。  

这正是我们保存信息的根本原因，不是吗？我们知道记忆脆弱，因此用技术作为保险。  

与其假装记住本书内容，不如假设读完几小时或几天后，它只会留下模糊印象。  

唯一留存的是你因本书建议而养成或改变的习惯。  

通过观察无数人实践 PARA 的过程，我总结了三个囊括全书精髓的习惯。每个习惯都基于永恒原则，即使技术变革仍将适用。它们将确保你不仅实现组织，更能长期保持。  
1. 按成果组织
2. 适时组织
3. 保持非正式

### 2.1. 习惯 1：按成果组织  

组织最大的陷阱之一是将其视为目的本身。  

整齐排列的工作空间或 Instagram 风格的精美笔记固然令人满足——如果你享受这些，本无可厚非。但我猜你拿起本书并非为此。  

PARA 的设计正是为了你的核心目标：  
- 每个决策都围绕“如何推动进展？”  
- 省去繁琐的标签、分类等前期工作。  
- 无需维护，只需完成项目。  
- 严格守护“项目”这一最行动导向的类别，排除看似有趣却无用的信息。  

这些设计旨在助你剔除一切无关目标的事物。  

无论你正冲刺项目终点，还是力求提升生活某领域的标准，始终以终为始，反向决定所需信息，其余一概舍弃。  

### 2.2. 习惯 2：适时组织  

我的组织哲学是：尽可能少组织、尽可能晚组织，且仅组织绝对必要部分。  

在一本关于组织的书中说这话或许奇怪，但组织本身不创造价值。除非它能让你进入高效行动的状态，否则毫无意义。  

因此 PARA 是极简主义方案——它仅在需求变化时轻推信息转移。这是一种“自下而上”的方法，有机响应生活变化。  

与其“以防万一”耗费精力组织数字信息，不如待需求明确时“适时”整理。这能避免前期无效劳动，将精力留给明确目标。  

### 2.3. 习惯 3：保持非正式  

> [!NOTE] Title
> PARA 仅在一个地方需要精确：项目定义。其余部分不仅允许一定混乱，更应如此。高度精确的系统需要大量维护精力，因此数字世界的多数领域默认应保持松散与非正式。  

这条规则承认：过度组织信息未必提升其价值。有时，过度组织反而阻碍有创意的连接。允许一定混乱，能为不同想法的碰撞创造机会。  

因此，我不认同许多组织专家的建议，例如：  
- 在文件夹内创建复杂结构  
- 为笔记或文档使用标准化模板  
- 创建多层嵌套子文件夹  
- 用数据库等正式方法管理个人信息  

我尝试过这些方法，但发现它们占用了本应用于思考的时间。请抵制创建复杂机制的冲动，简单直接的方式足矣。未来的你会感谢此刻省下的精力。  

### 2.4. 保护创意，静待花开  

新想法初生时极为脆弱，像婴儿般潜力巨大，却需抵御自我怀疑、他人批评、“不够好”的恐惧等风险。此时它无法独立存活，但这不意味它是坏点子——它只需时间和空间成长，正如人类。 

上述每个习惯都为新想法的萌发创造环境：  
- **按成果组织**确保你在现实中测试想法。  
- **适时组织**节省精力以抓住意外机遇。  
- **保持非正式**允许新奇连接与模式形成。  

将 PARA 的文件夹视为保护雏形想法的游乐场。文件夹的“墙”内是混乱的沙盒，松散相关的想法在此混合嬉戏，直至成熟到能独立面对世界。  

## 3. 用 PARA 提升专注力、创造力与全局观  

PARA 可助你培养知识工作者最渴望的三种心智状态：**专注**、**创造力**与**格局感**。  

### 3.1. 如何用 PARA 提升专注  

专注的本质是“一次只做一件事”。理论上简单，实践却非易事。  

我注意到，许多人试图在接收新信息的环境中专注：  
- 用收件箱当待办清单，却因不断涌入的邮件分心。  
- 用浏览器多任务处理，结果一天结束仍面对数十个未读标签页。  

要专注，需远离网络的喧嚣，退入私密空间。  

如果你有林间小木屋，我着实羡慕。对多数人而言，私密空间只需是另一个软件。  

当我需要专注时，会关闭 Wi-Fi。若任务所需文件全在云端，你永远无法脱离浏览器窗口外的信息海洋。每次坐下工作时，我会从待办清单选择任务，开启设备飞行模式，直至完成才重新联网。  

但若没有可信赖的全盘信息管理系统，这几乎无法实现。PARA 正是你的林间木屋——在此隔绝世界，打磨自己的创意、理论与成果，再将其带回现实。  

### 3.2. 如何用 PARA 培养创造力  

你可能多次听说创造力对现代职业成功的重要性。  

收集信息容易，归档也不难。但若止步于此，所有努力只是囤积。价值不来自输入，而来自带有你印记的输出。  

我在《打造第二大脑》中基于 PARA 构建了数字时代的创造力系统。但就本书而言，PARA 旨在创造创造力萌芽的环境。  

当你有大量精心筛选、与单一项目或目标相关的想法集中于一处时，奇迹便开始发生。  

利用这一点，建议根据共鸣筛选 PARA 的保存内容：  
- 什么打动了你？  
- 什么让你起鸡皮疙瘩、心跳加速或充满惊叹？  

这些是创造力的原料，值得保存。  

### 3.3. 如何用 PARA 培养格局感  

不妨将 PARA 的每个主类别视为一种“视野”：  
- **项目**属于短期视野，在未来几小时或几天内展开。  
- **责任领域**与**资源**属于中期视野，跨越数周或数月。  
- **存档**更可能在数月或数年的长期视野中发挥作用。  

按时间尺度划分信息有何用处？因为每个尺度需要截然不同的思维方式。  

**日常层面**，你应聚焦活跃项目。项目包含未来几小时或几天需采取的行动或查阅的信息。此阶段变化迅速，因此需让这些信息触手可及、占据注意力中心。  在这个短期视野中，相关问题包括：  
- 哪些项目当前最活跃？  
- 哪些任务最紧迫？  
- 推动进展的下一步是什么？  
- 需要哪些信息支持？

**责任领域**与**资源**在数周至数月的中期视野中相关。平日救火时或许无需频繁查阅，但进行每周回顾等深度思考时，它们便显现价值。

在进行更深入的反思时，问自己这些问题：
- 我对每个责任领域的标准（质量或表现）是什么？  
- 当前是否达标？  
- 若未达标，可启动、停止或改变哪些项目、习惯、例行事项？  
- 哪些资源能支持这些改变？  

评估资源时，可问：  
- 是否有新兴趣或热情值得深入？  
- 是否有新好奇心或问题值得探索？  
- 是否有停滞的爱好值得重启？  

通过按需求时间划分信息，PARA 让你在对应视野中获得恰当的格局感，助你跨时间尺度协同塑造理想未来。  

## 4. 若遇困惑，重新开始  

关于如何让 PARA 为你效力，我已给出诸多建议。本章将提供最后一条：  

**若卡住或感到不堪重负，只需按第三章指南归档一切，重新开始。**

例如：  
- 若文档文件夹沦为数字垃圾场，将所有内容移入带日期的存档，重新开始。  
- 若笔记应用收件箱堆积如山，将其移入带日期的存档文件夹，一劳永逸。  
- 若云存储盘混乱不堪，将所有内容移入带日期的存档，为新的一周、月份或年份重置。  

财务破产后果严重，但数字世界毫无代价。归档一切没有损失，因为所有内容未来仍可访问。  

### 4.1. 一切皆可选  

本书描述了我维护 PARA 系统的具体流程，但必须强调：所有步骤皆可取舍。没有一步是获取数字组织价值的绝对必需。  

- **某些周**太忙，无法整理收件箱或添加标题？那就不做。你知道总能用搜索找到所需，无需手动努力。  
- **某些周**太忙，无法逐项归类？全选后随便扔到某处即可。搜索仍是找回内容的魔法。  
- **某些时期**太疯狂，数周未更新项目清单？没关系。我保证，几个休眠项目不会引发灾难。  

PARA 极具韧性，它会按你的离开时的状态保存一切，直到你准备好回归——无论是七天后还是七个月后。这正是简单系统的意义：轻松抵御时间流逝。  

### 4.2. 放下对信息的执念  

多数人对数字财产有种责任感。从小父母教导我们爱护物品，这几乎成了一种道德义务——仿佛必须像整理物理环境一样精心维护数字环境。  

但这种态度在数字世界毫无意义。试图保存每一比特数据，如同囤积狂保留旧饰品和披萨盒。这些内容大多未经我们同意便出现，因此也可未经留恋地归档。  

我知道归档一切重新开始令人恐惧。你可能花数年思考如何组织这些文件，而我此刻建议你全数丢弃？是的，正是如此。  

**记住**：你并未失去任何东西。若真需要过去的内容，随时可深入存档复活它。但我猜，实际需求会远低于你的想象。  

真相是：搜索技术逐年进步，未来你最可能通过日益先进的算法访问文件。人工智能即将普及，未来你或许只需让 AI 搜索所有旧信息即可。因此，任何精心组织的努力终将白费。  

这是你划清界限、告别与信息的旧关系的良机。  

是时候拥抱新身份——成为拒绝琐碎细节、拥抱当下鲜活事物的人。  

你愿意接受我的邀请吗？  

## 5. 组织即个人成长  

在我的教练工作中，常听到客户以某种形式表达：“我就是没条理。”  

奇怪的是，说这话的往往是能力最强、效率最高、成就最多的人。若非渴望成就更多，“没条理”根本不成问题！  

我们似乎潜意识相信：“只要有条理，我就能成就任何事。”但深究之下，无人真正明白“有条理”的涵义。它如海市蜃楼，永远在远方诱惑，即便靠近，仍遥不可及。  

我想分享我对“有条理”的定义——首先谈谈它不是什么：  
- **有条理≠美观**：别被极简主义工作空间的假象迷惑，整洁的白空间不会赐你清晰或平和。  
- **有条理≠控制**：别试图通过 rigid 控制数字环境逃避生活固有的不确定性。  

人们常将 PARA 设为“理想生活”的镜像，而非反映真实生活。别用一堆空想的项目目标自欺。当你有勇气坦诚当下真正占据注意力的事物，并让 PARA 如实反映时，力量才开始流动。  

**你无法改变任何事，除非先彻底诚实面对需要改变的部分。**  

### 5.1. 智慧工作者的时代  

数十年来，我们自称“知识工作者”，因知识是核心资产。  

该术语诞生 60 多年后，知识工作者时代终于落幕。知识已通过搜索引擎普及，并随人工智能进步触手可及。知晓特定知识不再构成优势。  

我们正步入**智慧工作者时代**。  

如今重要的是：  
- 超越日常喧嚣，保持冷静格局。  
- 从此处协调身边涌动的人、系统、工具与信息流。  

现实如混沌的信息流无时无刻环绕我们。它不会以预制单元形式抵达，贴上“项目”或“责任”标签。**你**是主动雕琢信息块的主人，选择抓住令你触动的、鲜活的部分。  

许多人在使用 PARA 时意识到：自己早已拥有足够知识去追逐梦想。  

当你看到所有积累的有价值的材料，无论是自身或他人经验集中于一处时，就能得出结论：**你已准备就绪**。  

你的那个梦想是什么？你告诉自己“一个月/一年/三年/五年后就绪”的事，若诚实面对，是否其实今天就能开始？  

PARA 的承诺是：让“有条理”成为一件可快速完成的小事。它是从屏幕后踏入充满可能性的世界的踏板，是调动周身力量以实现人生使命的钥匙。  

# 附加内容：PARA 常见问题解答  

## 1. 项目

1. **如何组织单个项目？**  
   对于个人信息的组织，我不建议在单个项目内对笔记、文件等内容进行复杂分类。时间和精力应优先用于推进项目本身。  
   我的建议是：**保持项目足够小，无需内部结构**。若项目膨胀到需要额外管理，就应拆分。通常每个项目只需保存 5-10 项内容，按创建时间排序即可形成“时间线”，轻松推动微型项目完成。  
2. **如何处理暂时搁置的项目？**  
   项目列表的常见陷阱是混杂非活跃项目。这些项目会逐渐让列表变得沉重且脱离实际。  
   解决方案：**将非活跃项目移至存档**。存档并非单向坟墓，未来仍可随时重启。建议每周检查存档，发掘可复用的旧材料。  
3. **一个人同时进行多少项目最合适？**  
   大多数人适合同时管理 **10-15 个项目**。每周实际推进 3-5 个，其余处于等待或监督状态。数量需根据个人专注力调整，核心是严格区分“可行动”与“空想”。  
4. **如何存储未来项目？**  
   严格来说，**“未来项目”并不存在**。无目标、无期限、无进展的只是愿望。但可在笔记中创建“未来项目灵感”列表，待条件成熟时升级为正式项目。  

## 2. 领域

1. **责任区域应细化到什么程度？建议设置多少个？**  
   细化程度因人而异。例如“财务”可拆分为“税务”“储蓄”“投资”等。我本人有约 12 个工作相关和 12 个生活相关的责任区域。  
   关键原则：**细化到能明确评估是否达标**。例如将公司业务拆分为部门职能，便于针对性改进。 
2. **责任区域的笔记中应放什么内容？**  
   PARA 的核心是**分离“可行动”与“非行动”信息**。若将所有内容嵌套在责任区域下，会导致关键信息被淹没。  
   例如：学校“教育活动”文件夹中，只有少数文件与下周的新课程（项目）直接相关。与其在层级中费力筛选，不如将项目文件集中存放。  
3. **为何不应将所有“健康”相关内容集中存放？**  
   单一文件夹会迅速混乱，迫使你建立复杂子文件夹或标签系统。PARA 的解决方法是：  
	- **按时间尺度分离**：短期（项目）、中期（责任区域/资源）、长期（存档）  
	- **按平台分离**：文本（笔记应用）、图片（相册）、PDF（文档夹）  
	- 例如：  
		- **项目**：找新医生、报名健身房  
		- **责任区域**：运动笔记、医疗报告  
		- **资源**：健康食谱、习惯技巧  
   跨类别链接（如食谱链接到预算）可增强灵活性，但需避免过度链接导致拖延。  

## 3. 资源

1. **新资源文件夹从何而来？**  
   资源源自兴趣、灵感或潜在资产（如客户评价、代码片段）。可自由创建，无需限制数量。  
2. **如何跨领域和资源“交叉激发”创意？**  
   **链接功能**是关键！在笔记应用中关联相关内容，但需警惕“过度连接陷阱”——真正的价值在于完成并分享成果，而非构建复杂网络。  

## 4. 存档

1. **如何整理存档？**  
   存档无需精细管理。默认按字母排序文件夹，内部文件按创建时间倒序排列（自动完成）。  
2. **何时将内容移入存档？**  
	- **责任区域存档时机**：关系结束、搬家、职责变更等  
	- **资源存档时机**：兴趣消退、趋势过时、技术迭代  
	- 存档内容仍可随时搜索和调用，定期清理并复用旧知识。

## 5. 其他

1. **若不同平台的文件夹逐渐不一致？**  
   允许轻度差异，待严重影响效率时再花 5-10 分钟统一命名。  
2. **是否需要多个 PARA 系统？**  
   可分开工作与个人 PARA（如公司设备限制），但需明确边界。  
3. **移动文件夹时是否需要重命名？**  
   视情况而定。单个文件直接移动，整文件夹可重命名升级为项目（如“设计模板”→“打包发布模板项目✨”）。  
4. **大型项目如何管理？**  
   拆分为可执行的微型项目。例如写书→“完成第一章大纲”，创业→“寻找前三位客户验证需求”。  
5. **PARA 不适用哪些场景？**  
   需专业流程的领域（如医疗病历、科研数据）应使用专用工具。PARA 适用于开放式的日常管理。  
6. **如何用 PARA 整理实体物品？**  
- 按季节轮换衣物位置  
- 厨房 pantry 分层：常用调料放中层，派对餐具放顶层  
- 文件柜：常用文件近处，旧税存储在储物室
