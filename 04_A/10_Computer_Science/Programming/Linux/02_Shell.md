## 1. 终端 Terminal

```shell
fangxiang@fuck:~$
用户名@计算机名:当前所在目录$
```

- `$` 指的是等待用户输入
- `~` 指的是用户home目录
- `/` 指的是根目录
- `.` 单点符，当前目录
- `..` 双点符，当前目录的父目录

## 2. 文件目录

> [!NOTE] Linux 中一切皆文件

目录本身也是一种特殊的文件。

- 绝对路径

```bash
gedit /home/fangxiang/Documents/doc/1.txt
```

- 相对路径

```bash
gedit Documents/doc/1.txt
gedit ./Documents/doc/1.txt
gedit ~/Documents/doc/1.txt
```

## 3. Linux 根目录

- bin - 二进制目录，GNU工具，ls等自带的命令，存放许多用户级的目录
- boot - 启动目录
- cdrom - 光盘
- dev - 设备目录
- etc - 系统配置文件目录
- home - 主目录，显示所有用户目录
- lib, lib32, lib64, libx32 - 库目录
- lost+found - 断电、临时保存的文件目录
- media - 媒体目录
- mnt - 主要的挂载目录，外设和电脑进行连接
- opt - 可选目录
- proc - 伪文件系统
- root - root用户主目录
- run - 运行目录
- sbin - 系统二进制目录，GNU高级管理员使用的命令工具
- snap - Ubuntu产物
- srv - 服务目录，本地服务
- swapfile, sys
- tmp - 临时目录（windows也有，temp）
- usr - 用户二进制目录，GNU工具
- var - 可变目录
- FHS - 文件系统层级标准（Linux发行版）

## 1. 选项

```bash
-R     # 递归列出遇到的子目录。
-F     # 每个目录名加 "/" 后缀，每个 FIFO 名加 "|" 后缀， 每个可运行名加“ * ”后缀。
-r     # 逆序排列。
-a     # 列出所有文件，包括以 "." 开头的隐含文件。
-t     # 按时间信息排序。
-1     # 单列输出。
-A, --almost-all
       # 显示除 "." 和 ".." 外的所有文件。
-m, --format=commas
       # 水平列出文件，每行尽可能多，相互用逗号和一个空格分隔。
```

## 2. 实例

```bash
$ ls       # 仅列出当前目录可见文件
$ ls -CF   #缩写是 l
$ ls -l    # 列出当前目录可见文件详细信息
$ ls -hl   # 列出详细信息并以可读大小显示文件大小
$ ls -al   # 列出所有文件（包括隐藏）的详细信息
**$ ls -alF  #缩写是ll**
$ ls --human-readable --size -1 -S --classify # 按文件大小排序
$ du -sh * | sort -h # 按文件大小排序(同上)
$ ls --color=auto # 列出文件并标记颜色分类
```

## 4. ls进阶用法通配符

在Linux中，通配符 `?` 和 `*` 常用于文件名匹配，尤其是在命令行中处理文件时。它们允许你使用简洁的方式来匹配多个文件名。以下是它们的作用及搭配 `ls` 命令的用法：

### 4.1. `?` 通配符

`?` 通配符代表任意单个字符。它可以匹配文件名中的任何一个字符，但只占一个字符的位置。

---

假设当前目录下有以下文件：

```
file1.txt
file2.txt
file3.txt
file10.txt
```

使用 `ls` 命令查看所有包含一个字符变化的文件名：

```
ls file?.txt
```

这里，`file?.txt` 表示匹配任何一个字符，正好匹配 `fi·le1.txt` , `file2.txt` , `file3.txt` ，但 `file10.txt` 不会被匹配，因为它有两个字符的变化。

### 4.2. `*` 通配符

`*` 通配符表示**任意长度的字符（包括0个字符）**。它可以匹配文件名中的任意数量的字符，包括没有字符的情况。

---

假设当前目录下有以下文件：

```
report.doc
report1.doc
report2.doc
report_final.doc
```

使用 `ls` 命令查看所有以 `report` 开头的 `.doc` 文件：

```
ls report*.doc
```

这里，`report*.doc` 匹配所有以 `report` 开头、并以 `.doc` 结尾的文件名，无论中间有多少字符。

```
report.doc
report1.doc
report2.doc
report_final.doc
```

### 4.3. `[]` 通配符

#### 4.3.1. 方括号 `[` 和 `]` 的作用

1. **匹配一组字符**：
    - 方括号内的字符表示可以匹配文件名中**某一位置上的任何一个字符**。
    - 如果在方括号中列出多个字符，则表示可以匹配这些字符中的任意一个。
    **示例**：
    - `ls file[123].txt`
        - 匹配的文件名可以是 `file1.txt`，`file2.txt`，或 `file3.txt`，但不会匹配 `file4.txt` 或其他名字。
2. **匹配字符范围**：
    - 方括号内还可以使用连字符（``）来表示**字符范围**，如 `[a-z]` 表示从 `a` 到 `z` 的所有小写字母。
    **示例**：
    - `ls file[a-c].txt`
        - 匹配 `filea.txt`、`fileb.txt`、`filec.txt`，但不匹配 `filed.txt`。
3. **匹配数字范围**：
    - 与字母类似，方括号中也可以定义**数字范围**，如 `[0-9]` 代表 0 到 9。
    **示例**：
    - `ls report[0-9].txt`
        - 匹配 `report1.txt`, `report2.txt`，...，`report9.txt`，但不会匹配 `report10.txt`。
4. **否定字符集（`^` 或 `!`）**：
    - 在方括号的开头使用 `^`（有时也用 `!`）可以表示**非**，即不匹配方括号中列出的字符或字符范围。
    **示例**
    - `ls file[^0-9].txt`
        - 匹配所有文件名中除了数字字符的 `file.txt` 文件，换句话说，不匹配 `file0.txt` 到 `file9.txt` 的文件。

#### 4.3.2. 方括号的应用示例

1. **匹配指定字符集**：
    - 如果你想列出所有文件名以 `a`、`b` 或 `c` 结尾的 `.txt` 文件，可以使用： 这将匹配所有以 `a`、`b` 或 `c` 结尾的 `.txt` 文件，例如：
	
	```
	ls *[abc].txt
	
	```
	
	```
	filea.txt
	fileb.txt
	filec.txt
	
	```
	
2. **匹配字符范围**：
    
    - 假设有一组文件名如下：
        
        ```
        file1.txt
        file2.txt
        file3.txt
        filea.txt
        fileb.txt
        filec.txt
        
        ```
        
    - 如果你想匹配所有以 `1` 到 `3` 或 `a` 到 `c` 结尾的文件，可以使用： 结果将会是：
        
        ```
        ls *[1-3a-c].txt
        
        ```
        
        ```
        file1.txt
        file2.txt
        file3.txt
        filea.txt
        fileb.txt
        filec.txt
        
        ```
        
3. **否定匹配**：
    
    - 假设你有一组文件：
        
        ```
        report1.txt
        report2.txt
        report3.txt
        reportA.txt
        reportB.txt
        
        ```
        
    - 如果你想匹配所有不以数字结尾的 `.txt` 文件，可以使用： 结果将是：
        
        ```
        ls report[^0-9].txt
        
        ```
        
        ```
        reportA.txt
        reportB.txt
        
        ```

#### 4.3.3. 方括号与其他通配符的结合

可以结合使用 `?`, `*`, `[]` 等多个通配符来进行更复杂的文件匹配。例如：

- `ls file[1-3]?.txt` 将匹配 `file1a.txt`, `file2b.txt`，但不会匹配 `file10.txt`。
- `ls file[!0-9]*.txt` 将匹配不以数字开头的文件名。

#### 4.3.4. 总结

- **`[abc]`**：匹配 `a`、`b` 或 `c`。
- **`[a-z]`**：匹配范围内的字符，例如 `a` 到 `z`。
- **`[^...]`**：匹配不在字符集中的字符。
- 可以通过结合 `ls` 和 `[]` 进行灵活的文件匹配操作，尤其在批量处理文件时非常方便。

这些通配符在 shell 中非常强大，常用于查找、删除或批量操作文件。
